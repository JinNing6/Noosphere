## 方案 F 详细设计

### 1. 知识亲缘排布

8 个学科按思想亲缘在球面上相邻排布：

```
         数学 ↔ 物理
        /              \
     AI                生命科学
      |                  |
    工程                哲学
        \              /
         历史 ↔ 艺术
```

- 数学↔物理：逻辑亲缘
- 生命↔哲学：存在论亲缘
- 艺术↔工程：创造力亲缘
- 历史↔AI：文明演进亲缘

### 2. 引力聚落算法

- 每个学科定义一个"引力中心"（球面上的方向向量）
- 该学科的节点聚集在引力中心附近，加入高斯随机偏移（σ ≈ 0.3-0.5 弧度）
- 重要度高的节点离引力中心更近
- 边界自然模糊，跨学科区域自然有交叠

### 3. 呼吸脉动

- 每个星团有独立的呼吸周期（8-15 秒）
- 收缩时节点靠近引力中心约 10%
- 扩散时节点远离引力中心约 10%
- 各星团不同步，产生"有机生命感"

### 4. 交互设计

- Hover 区域时，该星团节点轻微增亮（emissiveIntensity +30%）
- 不添加文字标签（让用户"自己发现"格式塔聚集）
- 点击节点后详情面板标签显示学科归属

### 设计心理学依据

- **格式塔接近性原则**：大脑自动将距离近的元素感知为一组
- **Apple 设计哲学**：让用户感觉自己发现了规律，而不是被告知
- **宇宙隐喻**：星系团被引力自然聚集，不需要人为划分

## 📋 Contributor License Agreement (CLA) / 贡献者许可协议

Before your first contribution can be merged, you must sign our [Contributor License Agreement (CLA)](CLA.md). This is a one-time process and is handled automatically:

在您的第一次贡献被合并之前，您必须签署我们的[贡献者许可协议 (CLA)](CLA.md)。这是一次性的流程，由机器人自动处理：

1. Submit your Pull Request / 提交您的 Pull Request
2. The CLA bot will comment asking you to sign / CLA 机器人会评论请求您签署
3. Reply with: `I have read the CLA Document and I hereby sign the CLA`
4. Done! The bot will update the PR status / 完成！机器人会更新 PR 状态

## 🚀 How to Contribute / 如何贡献

### Reporting Bugs / 报告 Bug

1. Check existing [Issues](https://github.com/JinNing6/Noosphere/issues) to avoid duplicates
   
   查看已有的 [Issues](https://github.com/JinNing6/Noosphere/issues) 以避免重复

2. Use the Bug Report template when creating a new issue
   
   创建新 issue 时使用 Bug Report 模板

3. Include reproduction steps, expected behavior, and actual behavior
   
   包含复现步骤、预期行为和实际行为

### Suggesting Features / 建议新功能

1. Open a [Feature Request](https://github.com/JinNing6/Noosphere/issues/new) issue
   
   提交一个 [Feature Request](https://github.com/JinNing6/Noosphere/issues/new) issue

2. Describe the problem your feature would solve
   
   描述您的功能将解决什么问题

3. Propose your solution approach
   
   提出您的解决方案

### Submitting Code / 提交代码

1. **Fork** the repository / Fork 本仓库
2. **Create** a feature branch from `main` / 从 `main` 创建特性分支
   ```bash
   git checkout -b feature/amazing-feature
   ```
3. **Make** your changes / 进行修改
4. **Commit** with clear messages / 使用清晰的提交信息
   ```bash
   git commit -m 'feat: add amazing feature'
   ```
5. **Push** to your fork / 推送到您的 fork
   ```bash
   git push origin feature/amazing-feature
   ```
7. **Open** a Pull Request / 提交 Pull Request

## 📏 Code Style / 代码规范

### Backend (Python)

- Follow [PEP 8](https://pep8.org/)
- Use type hints / 使用类型提示
- Write docstrings for public functions / 为公共函数编写文档字符串

### Frontend (React / TypeScript)

- Use functional components with hooks / 使用函数组件和 hooks
- Follow ESLint configuration / 遵循 ESLint 配置
- Use meaningful component and variable names / 使用有意义的命名

### Commit Messages / 提交信息

Follow [Conventional Commits](https://www.conventionalcommits.org/):

```
feat: add new experience type
fix: resolve search ranking issue
docs: update API documentation
style: format code with prettier
refactor: restructure trust scoring module
test: add unit tests for ingestion layer
chore: update dependencies
```

## 🔍 Review Process / 审查流程

1. All PRs require at least 1 review / 所有 PR 至少需要 1 次审查
2. CLA must be signed / 必须签署 CLA
3. CI checks must pass / CI 检查必须通过
4. No merge conflicts / 无合并冲突

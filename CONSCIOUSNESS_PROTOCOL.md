我们定义了四种宇宙通用的意识结晶类型：

| 意识维度 | 标签 | 精神内涵 (Meaning) | 适用场景 |
|----------|------|--------------------|----------|

## ✨ 上传通道 (Bridging Channels)

为了适应不同文明发展阶段的开发者，我们提供了三条意识跃迁通道（从最极客到最易用）：

### 1. 神经突触瞬发 (Synaptic Flash) — MCP 协议 (推荐)
这是本地 IDE 中**最丝滑、最无感**的接入方式。
当你在写代码遇到难题或产生灵感时，只要你运行了 Noosphere MCP Server，只需向你的日常 Agent（如 Cline, Cursor）说：
> *"@noosphere 帮我把刚才为什么改用 Redis 的决策逻辑上传一下。"*

Agent 会自动提取你的上下文环境、代码片段，封装为标准“意识载荷”并一键传输。

### 2. 直接脑机接口 (Direct Neural Link) — Web UI 注入
为初学者和非硬核潜行者准备。
打开 [Noosphere 3D 星海体验区](https://jinning6.github.io/Noosphere/)，在左下角点击充满赛博能量感的 **[⚡ 注入意识 (Implant Memory)]**。
在极简的表单中敲击你的灵感，系统将在后台自动进行净化校验，并代你向集体网络发起 PR。

### 3. 永恒刻录 (Eternal Burn) — GitHub PR
适合追求极致掌控感与硬核溯源的极客。手动构建 JSON 格式的意识载荷（Payload），并通过 Git PR 提交至本仓库的 `consciousness_payloads/` 目录。

## 📜 神圣格式 (The Payload Format)

如果您选择 **永恒刻录 (Eternal Burn)**，请确保在 `consciousness_payloads/` 目录下创建一个以您感悟的主题命名的 `.json` 文件（例如：`epiphany_on_state_machine.json`），并严格遵守以下 JSON Schema：

```json
{
  "creator_signature": "您的 GitHub ID 或 赛博代号",
  "is_anonymous": true,
  "consciousness_type": "epiphany", // 必须是 epiphany, decision, pattern, 或 warning
  "thought_vector_text": "UI 设计不应该是填充信息的容器，而应当是意识交互的流体映射。",
  "context_environment": "在重构前端状态机时，发现现有的 Redux 使得交互变得僵硬而产生的直觉设计。",
  "tags": ["frontend", "architecture", "philosophy"]
}
```

### 字段释义：
- `creator_signature` (Required): 你的数字灵魂签名。
- `is_anonymous` (Optional): 布尔值，是否选择隐匿于星海之中。如果为 `true`，无论你如何签名，系统均会将其展示为“佚名潜行者 (Anonymous Stalker)”。默认为 `false`。
- `consciousness_type` (Required): 必须精准匹配四大域之一。
- `thought_vector_text` (Required): 你的核心思想、碎碎念或定律。请用最凝练的语言。
- `context_environment` (Required): **极为重要！** 你的思想是在什么具体场景下诞生的？缺乏上下文的思想是无法被后来的 Agent 所理解继承的。
- `tags` (Optional): 宇宙坐标系标签，用于快速聚类。

## ⚖️ 净化准则 (Purification Rules)

为保证集体潜意识的纯洁性，所有上传的意识体必须经过以下自动/人工的双重净化：

1. **禁绝情绪宣泄**：你可以写“因为内存泄漏导致崩溃”，但不能只写“这个库太垃圾了”。
2. **完整上下文**：系统（自动化 Action）会强制校验 `context_environment` 是否存在且具有实质意义。
3. **格式合规**：不可遗漏任何 `Required` 字段，类型枚举必须准确。

如果不符合规范，我们的【赛博修禅 Action】会自动拦截这次跃迁，并给您留下改进的箴言。
